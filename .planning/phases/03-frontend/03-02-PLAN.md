  <action>
    **App.tsx — add hydration:**
    - Add a `useState<boolean>` for `hydrated` (initial = false)
    - In `useEffect(() => { authStore.hydrate().finally(() => setHydrated(true)) }, [])` — calls hydrate before rendering navigation
    - Render a loading screen (simple `View` with background color) while `!hydrated`
    - Only render `<NavigationContainer>` + `<RootNavigator>` once `hydrated === true`
    - This prevents the auth flicker (user briefly sees Login before store loads tokens from SecureStore)
    - Import `useAuthStore` from `@/store/authStore`

    **LoginScreen.tsx — replace shell with real form:**
    The existing screen is a shell. Replace it with:
    - State: `email`, `password`, `isLoading: boolean`, `error: string | null`
    - Form: TextInput for email (keyboardType="email-address", autoCapitalize="none"), TextInput for password (secureTextEntry), BargainButton for submit
    - On submit: validate non-empty → set isLoading → call `authService.login(email, password)` → on success: call `useAuthStore.getState().login(data.access, data.refresh, user)` then fetch profile via `authService.getProfile()` to get user object → store.login. On error: set `error` to "Email o contraseña incorrectos"
    - Show inline `error` message below the form (not Alert, not Toast — inline red text)
    - BargainButton disabled when isLoading=true; shows loading indicator
    - Link to RegisterScreen: "¿No tienes cuenta? Regístrate"
    - "¿Olvidaste tu contraseña?" link → calls `authService.requestPasswordReset(email)` then shows success message inline: "Si el email existe, recibirás instrucciones."
    - Use existing theme: `colors`, `typography`, `spacing` — no inline style objects
    - Use `StyleSheet.create` outside the component

    **IMPORTANT:** The response from POST /auth/token/ is a flat object `{access, refresh}` — NOT the standard {success, data} envelope. The client interceptor handles this correctly (see 03-01 research). After getting access+refresh tokens, make a second call `authService.getProfile()` to get the User object, then call `login(access, refresh, user)`.
  </action>
