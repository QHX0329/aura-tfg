    **Step 3 — Upgrade frontend/src/api/client.ts** with the full refresh-and-retry pattern. KEEP the existing base URL and headers. ADD:
    - Module-level `let isRefreshing = false` and `let refreshQueue: Array<(token: string) => void> = []`
    - Upgrade response success interceptor: check `if (response.data?.success !== undefined) return response.data.data; return response.data;`
    - Upgrade response error interceptor: on 401 + !_retry, check isRefreshing flag. If refreshing → queue the request. If not refreshing → set flag, read refresh_token from SecureStore, call POST /auth/token/refresh/, update store + SecureStore, drain queue, retry. On refresh failure → call logout().
    - Import SecureStore from 'expo-secure-store'
    - The raw axios.post for /auth/token/refresh/ must use a plain `axios.create()` (not apiClient) to avoid interceptor recursion

    **Step 4 — Upgrade frontend/src/store/authStore.ts:** Add:
    - `refreshToken: string | null` field to AuthState interface
    - `memberSince: string | null` field (populated from backend user.date_joined)
    - `setRefreshToken: (token: string) => void` action
    - `hydrate: () => Promise<void>` action — reads 'access_token', 'refresh_token', 'auth_user' from SecureStore; if tokens exist, calls setToken + setRefreshToken + setUser + sets isAuthenticated=true
    - Update `login(token, refreshToken, user)` signature — writes both tokens to SecureStore, also writes user as JSON under 'auth_user'
    - Update `logout()` — deletes all three SecureStore keys
    - NOTE: Zustand 5 does NOT support built-in persist middleware with SecureStore. Use manual hydration — no middleware needed.

    **Step 5 — Create __tests__/apiClient.test.ts** as the Wave 0 test scaffold. Mock expo-secure-store and axios. Test the behaviors listed in the behavior block above. Tests should be RED before Step 3 is complete.

    **API Service Modules** — each in `frontend/src/api/`. All import from `./client` and use `apiClient`. Never call axios directly.

    `authService.ts`:
    - `login(email, password)` → POST /auth/token/
    - `register(data: {email, password, first_name, last_name})` → POST /auth/register/
    - `refreshToken(refresh: string)` → POST /auth/token/refresh/ (used internally by interceptor only)
    - `getProfile()` → GET /auth/profile/me/
    - `updateProfile(data)` → PATCH /auth/profile/me/
    - `updatePreferences(prefs)` → PATCH /auth/profile/me/preferences/
    - `changePassword(oldPassword, newPassword)` → PATCH /auth/profile/me/ with {old_password, new_password}
    - `requestPasswordReset(email)` → POST /auth/password-reset/

    `listService.ts`:
    - `getLists()` → GET /lists/
    - `getList(id)` → GET /lists/{id}/
    - `createList(name)` → POST /lists/
    - `updateList(id, data)` → PATCH /lists/{id}/
    - `deleteList(id)` → DELETE /lists/{id}/
    - `addItem(listId, {product_id, quantity})` → POST /lists/{id}/items/
    - `updateItem(listId, itemId, data)` → PATCH /lists/{id}/items/{itemId}/
    - `deleteItem(listId, itemId)` → DELETE /lists/{id}/items/{itemId}/

    `productService.ts`:
    - `search(query: string, page?: number)` → GET /products/?search={query}
    - `autocomplete(q: string)` → GET /products/autocomplete/?q={q}

    `storeService.ts`:
    - `getNearby(lat: number, lng: number, radius?: number)` → GET /stores/?lat={lat}&lng={lng}&radius={radius}

    `notificationService.ts`:
    - `getNotifications(page?: number)` → GET /notifications/?page={page}
    - `markAsRead(id: string)` → PATCH /notifications/{id}/read/
    - `markAllAsRead()` → PATCH /notifications/ with {mark_all_read: true}
    - `deleteNotification(id: string)` → DELETE /notifications/{id}/
    - `registerPushToken(token: string, deviceId: string)` → POST /notifications/push-token/ with {token, device_id}

    `priceService.ts`:
    - `getPriceAlerts()` → GET /prices/alerts/ (active alerts for current user)
    - `getPriceComparison(productId: string)` → GET /prices/?product={productId}
