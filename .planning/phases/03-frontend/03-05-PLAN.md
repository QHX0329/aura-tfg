    **MapScreen.tsx — replace shell:**

    **Location permission flow (must be in this exact order):**
    1. On mount: call `Location.requestForegroundPermissionsAsync()`
    2. If `status !== 'granted'`: show a centered card "Activa la ubicación para ver tiendas cercanas" + TouchableOpacity "Abrir configuración" → `Linking.openSettings()`. Do NOT render MapView.
    3. If granted: call `Location.getCurrentPositionAsync({accuracy: Location.Accuracy.Balanced})`
    4. Dev mode (`__DEV__`): use hardcoded Seville coords (lat: 37.3886, lng: -5.9823) as fallback if location unavailable.

    **MapView setup:**
    ```tsx
    import MapView, { Marker } from 'react-native-maps';

    <MapView
      style={StyleSheet.absoluteFillObject}
      initialRegion={{
        latitude: userLat,
        longitude: userLng,
        latitudeDelta: 0.05,
        longitudeDelta: 0.05,
      }}
      showsUserLocation={true}
      showsMyLocationButton={true}
    >
      {stores.map(store => (
        <Marker
          key={store.id}
          coordinate={{ latitude: store.location.lat, longitude: store.location.lng }}
          title={store.name}
          description={store.address}
        />
      ))}
    </MapView>
    ```

    **Store data:** After getting user coordinates, call `storeService.getNearby(lat, lng, 10)`. Store results in local state. Show loading overlay (SkeletonBox covering the map area, borderRadius 0) while fetching.

    **Store list panel:** Below the map (or as a bottom sheet — Claude's discretion), show a horizontal FlatList of store cards. Same visual style as NearbyStoreCard sub-component from HomeScreen. Tap a store card → move map to that store's location (`mapRef.current.animateToRegion(...)`).

    **Note on backend coords:** The backend returns `location` as a PostGIS GeoJSON point: `{type: "Point", coordinates: [lng, lat]}`. Extract as `coords = store.location.coordinates; lat = coords[1]; lng = coords[0]`. Update the `Store` interface in domain.ts if needed to add `location?: {type: string; coordinates: [number, number]}`.

    **App.json:** Confirm `react-native-maps` is in the expo plugins section if needed for SDK 55 (it auto-configures without explicit plugin entry in most cases, but add the config key for Android maps API key per RESEARCH.md pitfall 3).
