    **Step 1 — Bootstrap Vite project (run from frontend/ directory):**
    ```bash
    cd frontend
    npm create vite@latest web -- --template react-ts
    cd web
    npm install antd axios react-router-dom
    npm install --save-dev @testing-library/react @testing-library/jest-dom vitest jsdom
    ```

    **Step 2 — vite.config.ts:** Add `server.watch.usePolling: true` for Windows filesystem compatibility (RESEARCH.md pitfall 7). Set `test` config for vitest.

    **Step 3 — frontend/web/src/api/client.ts (new file):**
    - Create axios instance with `baseURL: import.meta.env.VITE_API_URL || 'http://localhost:8000/api/v1'`
    - Request interceptor: reads JWT from `localStorage.getItem('access_token')`, attaches as `Authorization: Bearer {token}`
    - Response interceptor: same {success, data} unwrapping pattern as mobile client. On 401: clear localStorage and `window.location.href = '/login'`
    - Export `apiClient` and `API_BASE_URL`

    **Step 4 — frontend/web/src/store/businessStore.ts (new file — Zustand):**
    ```typescript
    import { create } from 'zustand';
    interface BusinessProfile {
      id: string; business_name: string; tax_id: string; address: string; website?: string;
      is_verified: boolean; verification_status: 'pending' | 'verified' | 'rejected';
      rejection_reason?: string; price_alert_threshold_pct: number;
    }
    interface Promotion {
      id: string; product: {id: string; name: string}; store: {id: string; name: string};
      discount_type: 'flat' | 'percentage'; discount_value: number;
      start_date: string; end_date?: string; is_active: boolean;
      min_quantity?: number; views: number;
    }
    interface BusinessStoreState {
      token: string | null;
      profile: BusinessProfile | null;
      promotions: Promotion[];
      isLoading: boolean;
      setToken: (t: string) => void;
      setProfile: (p: BusinessProfile) => void;
      setPromotions: (list: Promotion[]) => void;
      logout: () => void;
    }
    // Persist token to localStorage manually (no zustand-persist needed)
    ```

    **Step 5 — frontend/web/src/App.tsx:**
    - Use `react-router-dom` v6 with `BrowserRouter`, `Routes`, `Route`
    - Routes:
      - `/login` → `LoginPage` (public)
      - `/*` → Protected routes wrapped in `UnverifiedGuard`
        - `/` → redirect to `/dashboard`
        - `/dashboard` → `DashboardPage`
        - `/prices` → `PricesPage`
        - `/promotions` → `PromotionsPage`
        - `/profile` → `BusinessProfilePage`
    - Auth guard: check `localStorage.getItem('access_token')` — if not present, redirect to `/login`
    - On app init: if token exists in localStorage, restore it to businessStore

    **Step 6 — frontend/web/src/components/AppLayout.tsx:**
    Use Ant Design `Layout` with `Sider` (collapsed on mobile breakpoint) + `Content`:
    - Sider: Ant Design `Menu` with items: Dashboard, Precios, Promociones, Perfil. Each menu item navigates via `useNavigate`.
    - Header: "BargAIn Business" title + logout button (right side)
    - Logout: clears localStorage, calls businessStore.logout(), redirects to /login

    **Step 7 — frontend/web/src/components/UnverifiedGuard.tsx:**
    ```tsx
    // Checks profile.verification_status:
    // - if 'pending': show full-page AntD Result component ("Tu solicitud está siendo revisada...")
    // - if 'rejected': show full-page AntD Result with rejection reason + Edit Profile button
    // - if 'verified': render children
    // - if profile is null (loading): show AntD Spin centered
    ```

    **Step 8 — frontend/web/src/pages/LoginPage.tsx:**
    - Ant Design `Form` + `Form.Item` + `Input` + `Input.Password` + `Button`
    - On submit: POST to `/auth/token/` using plain axios (not apiClient, to avoid interceptor auth cycle). Store `access` token in localStorage + businessStore. Then GET `/business/profiles/` to load the business profile into businessStore. Navigate to `/`.
    - On error: Ant Design `message.error('Credenciales incorrectas')`
    - Centered card layout (Ant Design `Card`) with BargAIn Business title

    **DashboardPage.tsx:**
    Use Ant Design `Row` + `Col` with `Statistic` components:
    - On mount: GET `/business/profiles/` and GET `/business/promotions/?is_active=true` and GET `/business/prices/?limit=5&ordering=-updated_at`
    - Stats cards: (1) Nombre del negocio (text), (2) Promociones activas (count), (3) Últimas actualizaciones de precio (count). Use `Statistic` component.
    - Recent price updates: `List` component showing last 5 price records (product name + price + date)
    - NOTE per CONTEXT.md decision: no competitor price alert banner in the portal (email-only)

    **PricesPage.tsx:**
    - Ant Design `Table` of current business prices — columns: Product, Price, Unit Price, Updated At, Actions
    - "Añadir precio" button → opens `Drawer` (right side) with a form:
      - Product search: Ant Design `AutoComplete` calling GET `/products/?search={q}` on type
      - Barcode scan: button "Escanear código de barras" → trigger `navigator.mediaDevices.getUserMedia({video: {facingMode: 'environment'}})` → feed to `<video>` element → use `BarcodeDetector` API if available; if not → show `Input` fallback for manual barcode entry. On barcode detected: GET `/products/?barcode={code}` to look up product.
      - Price `InputNumber` + unit price `InputNumber` + optional offer price + offer end date `DatePicker`
      - Submit: POST or PATCH `/business/prices/`
    - Edit action in table row → same Drawer with pre-filled values

    **PromotionsPage.tsx:**
    - Ant Design `Table` of promotions — columns: Product, Store, Discount, Dates, Status (active/expired), Views, Actions
    - "Nueva promoción" button → opens `Modal` with form:
      - Product: `AutoComplete` (same pattern as PricesPage)
      - Store: read-only (auto-filled from businessStore.profile)
      - Discount type: `Select` (flat / percentage)
      - Discount value: `InputNumber`
      - Start date: `DatePicker`
      - End date: `DatePicker` (optional)
      - Min quantity: `InputNumber` (optional, default 1)
      - Title and description: `Input` and `TextArea`
    - Submit: POST `/business/promotions/`
    - Deactivate button per row: PATCH `/business/promotions/{id}/` with `{is_active: false}` — available to both business and admin roles

    **BusinessProfilePage.tsx:**
    - Ant Design `Descriptions` for read-only view: Business name, CIF, address, website, verification status
    - Verification status: Ant Design `Tag` — green "Verificado", orange "Pendiente", red "Rechazado"
    - "Editar" button → switches to `Form` edit mode (or opens a `Modal`)
    - Editable fields: business_name, address, website (CIF is read-only after submission per backend)
    - Submit: PATCH `/business/profiles/{id}/` → refresh profile in businessStore
