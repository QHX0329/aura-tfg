  <files>backend/apps/stores/views.py, config/settings/base.py, backend/tests/unit/test_stores.py, backend/tests/integration/test_store_endpoints.py</files>
  <behavior>
    - Test 1: GET /stores/{id}/places-detail/ with store that has google_place_id returns {opening_hours, rating, user_rating_count, website_url}
    - Test 2: GET /stores/{id}/places-detail/ with store that has NO google_place_id returns {}
    - Test 3: Second call to same endpoint returns cached data (mocked Google API called only once)
    - Test 4: When GOOGLE_PLACES_API_KEY is empty, returns {}
    - Test 5: When Google API returns error, returns {} (silent fail)
    - Test 6: Unauthenticated request returns 401
  </behavior>
  <action>
    1. In `config/settings/base.py`, add: `GOOGLE_PLACES_API_KEY = config("GOOGLE_PLACES_API_KEY", default="")` (python-decouple is already used in the project).

    2. In `backend/apps/stores/views.py`, add a new `@action` method `places_detail` on `StoreViewSet`:
       - Decorator: `@action(detail=True, methods=["get"], url_path="places-detail", permission_classes=[IsAuthenticated])`
       - Cache key format: `places_detail:{pk}`
       - Check `cache.get(cache_key)` first — return cached if exists.
       - Fetch `Store.objects.get(pk=pk, is_active=True)`. If DoesNotExist, raise Http404.
       - If `store.google_place_id` is falsy, return `success_response({})`.
       - If `settings.GOOGLE_PLACES_API_KEY` is falsy, return `success_response({})`.
       - Call `requests.get(f"https://places.googleapis.com/v1/places/{place_id}", headers={"X-Goog-Api-Key": api_key, "X-Goog-FieldMask": "currentOpeningHours,rating,userRatingCount,websiteUri"}, timeout=5)`.
       - On ANY exception (network, 4xx, 5xx), return `success_response({})` — silent fail per user decision.
       - Normalize response: `{"opening_hours": data.get("currentOpeningHours"), "rating": data.get("rating"), "user_rating_count": data.get("userRatingCount"), "website_url": data.get("websiteUri")}`.
       - `cache.set(cache_key, normalized, timeout=60 * 60 * 24)` — 24h TTL.
       - Return `success_response(normalized)`.
       - Add `import requests as http_requests` (alias to avoid conflict with DRF `request`) and `from django.core.cache import cache` and `from django.conf import settings`.

    3. Add `@extend_schema` decorator for OpenAPI documentation with appropriate summary and response description.

    4. Write tests in `backend/tests/integration/test_store_endpoints.py`:
       - `test_places_detail_endpoint`: Create a store with google_place_id, mock `requests.get` to return Places-style JSON, assert 200 with normalized fields.
       - `test_places_detail_no_place_id`: Store without google_place_id returns `{"success": true, "data": {}}`.
       - `test_places_detail_unauthenticated`: Assert 401 for unauthenticated request.

    5. Write test in `backend/tests/unit/test_stores.py`:
       - `test_places_detail_cache_hit`: Mock `requests.get` + `cache`, call endpoint twice, assert `requests.get` called only once.

    6. Use `unittest.mock.patch` to mock `requests.get` in all tests — never call Google API in tests.
