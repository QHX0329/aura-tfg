    4. **services/distance.py** — `get_distance_matrix(points: list[tuple[float, float]], graphhopper_url: str = None) -> tuple[list[list[float]], list[list[float]]]`:
       - Default `graphhopper_url` from `settings.GRAPHHOPPER_URL` (add `GRAPHHOPPER_URL = env("GRAPHHOPPER_URL", default="http://graphhopper:8989")` to base.py)
       - POST to `{graphhopper_url}/matrix` with payload:
         ```python
         {"from_points": [[lng, lat] for lat, lng in points],
          "to_points": [[lng, lat] for lat, lng in points],
          "out_arrays": ["distances", "times"],
          "vehicle": "car"}
         ```
         **IMPORTANT:** Graphhopper expects `[lng, lat]` order, NOT `[lat, lng]`.
       - Parse response: `distances` (meters -> km by /1000), `times` (seconds -> minutes by /60)
       - Return `(distance_matrix_km, time_matrix_minutes)`
       - **Fallback:** If Graphhopper request fails (ConnectionError, Timeout, non-200), log warning with structlog and compute haversine distances using PostGIS `ST_Distance`:
         - For each pair of points, compute `GEOSGeometry(Point(lng, lat, srid=4326)).transform(3857)` distance
         - Estimate time as `distance_km / 40 * 60` (avg urban speed 40 km/h)
       - `timeout=10` on the HTTP request

    5. **services/solver.py** — `solve_route(...)` function:
       - Input: `distance_matrix: list[list[float]], time_matrix: list[list[float]], price_savings: list[float], weights: dict[str, float], max_stops: int`
       - Create `RoutingIndexManager(n, 1, 0)` — n locations, 1 vehicle, depot=index 0 (user location)
       - Create `RoutingModel(manager)`
       - Register arc cost callback per CLAUDE.md formula:
         ```python
         def arc_cost(from_idx, to_idx):
             from_node = manager.IndexToNode(from_idx)
             to_node = manager.IndexToNode(to_idx)
             dist_score = weights["distancia"] * distance_matrix[from_node][to_node]
             time_score = weights["tiempo"] * time_matrix[from_node][to_node]
             price_bonus = weights["precio"] * price_savings[to_node]
             return int((dist_score + time_score - price_bonus) * 1000)
         ```
       - Add stop count dimension: `routing.AddDimension(count_idx, 0, max_stops, True, "stop_count")`
       - Search params: `first_solution_strategy = PATH_CHEAPEST_ARC`, `time_limit.seconds = 5`
       - Extract solution route (list of node indices), return empty list if no solution

       Also create `optimize_shopping_list(shopping_list_id: int, user_lat: float, user_lng: float, max_distance_km: float, max_stops: int, weights: dict) -> dict`:
       - Query `ShoppingListItem.objects.filter(shopping_list_id=shopping_list_id, is_checked=False).select_related("product")`
       - Query candidate stores within radius using PostGIS: `Store.objects.filter(location__distance_lte=(user_point, D(km=max_distance_km)))`
       - If no stores: raise `StoreNotFoundError` with code `OPTIMIZER_NO_STORES_IN_RADIUS`
       - For each candidate store, query `Price.objects.filter(product__in=products, store=store, source="scraping")` to get latest prices
       - Filter to stores that carry at least 1 product from the list
       - Build points list: `[user_location] + [store.location for store in candidate_stores]`
       - Call `get_distance_matrix(points)`
       - Normalize: price_savings per store = sum of (max_price - store_price) for each product; normalize 0-1
       - Normalize distance and time matrices 0-1
       - Call `solve_route(dist_matrix, time_matrix, price_savings, weights, max_stops)`
       - Build route_data: ordered list of `{"store_id": ..., "store_name": ..., "chain": ..., "lat": ..., "lng": ..., "distance_km": ..., "time_minutes": ..., "products": [{"product_id": ..., "name": ..., "price": ...}]}`
       - Compute totals: `total_price`, `total_distance_km`, `estimated_time_minutes`
       - Return result dict
