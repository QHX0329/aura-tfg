## Spacing Scale

Declared values (all multiples of 4). Source: `frontend/src/theme/spacing.ts`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, chip internal gaps |
| sm | 8px | Gaps between inline elements, list item internal padding |
| md | 16px | Default component padding (cards, inputs, screen horizontal padding) |
| lg | 24px | Section padding, vertical spacing between card groups |
| xl | 32px | Spacing between major content blocks |
| 2xl | 48px | Header areas, generous section breaks |
| 3xl | 64px | Empty state vertical centering, onboarding hero spacing |

Exceptions:
- Touch targets (buttons, icon-only taps): minimum 44px height — use `sizes.searchBarHeight: 44` or `sizes.buttonHeight.md: 48` tokens from spacing.ts.
- Route step row: use `sizes.listItemHeight: 56` for map route stop rows.
- Bottom sheet drag handle area: 8px handle + 16px padding above first content element.

## Typography

Source: `frontend/src/theme/typography.ts`. Use `textStyles` composites — do not construct ad-hoc fontSize+fontWeight combinations.

**Scale: 4 sizes × 2 weights only.**

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Caption / label / bodySmall | 14px | 400 (regular) or 600 (semibold) | 1.55 | `textStyles.bodySmall` / `textStyles.label` |
| Body / button | 16px | 400 (regular) or 600 (semibold) | 1.55 / 1.35 | `textStyles.body` / `textStyles.button` |
| Heading / price | 20px | 600 (semibold) | 1.35 | `textStyles.heading3` / `textStyles.price` |
| Display / priceLarge | 28px | 600 (semibold) | 1.35 | `textStyles.heading1` / `textStyles.priceLarge` |

Token assignments for this phase:

| Token | Size | Weight | Notes |
|-------|------|--------|-------|
| `textStyles.body` | 16px | 400 | Default body text in bubbles, descriptions |
| `textStyles.button` | 16px | 600 | All CTA button labels |
| `textStyles.bodySmall` | 14px | 400 | Metadata, timestamps, OCR raw text (left column), confidence scores |
| `textStyles.label` | 14px | 600 | Savings badge label, OCR match label, route distance/time labels — weight contrast provides distinction from bodySmall, not size |
| `textStyles.caption` | 14px | 400 | Price-per-unit annotations, store distance labels — visual distinction via `textMuted` color, not a separate size |

## Color

Source: `frontend/src/theme/colors.ts`. The BargAIn palette is "Mercado Mediterráneo Digital" — warm white, cream surface, Triana orange primary.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAF7` (background) | Screen backgrounds, full-bleed areas |
| Secondary (30%) | `#FDF6EC` (surface) | Cards, bottom sheets, section containers, chat bubbles (assistant), OCR item rows |
| Accent (10%) | `#E8541A` (primary) | Reserved list below |
| Destructive | `#C0392B` (error) | Destructive actions only; error state banners |

Accent (`#E8541A`) reserved specifically for:
1. Primary CTA buttons ("Optimizar ruta", "Escanear lista", "Enviar mensaje")
2. Active tab indicator in bottom tab bar
3. "Ampliar radio" action button in OPTIMIZER_NO_STORES_IN_RADIUS error card
4. Loading progress indicators on the three wired screens (RouteScreen, OCRScreen, AssistantScreen) while requests are in flight
5. Confidence badge background when OCR match confidence >= 80%

Secondary semantic colors permitted in this phase:
- `#3A7D44` (success): savings summary row ("Ahorro total: X €"), optimal route badge
- `#D4850A` (warning): price freshness warning ("Precio actualizado hace más de 24h")
- `#1A6B8A` (info): non-critical assistant system messages
- `colors.chains.*`: store chain identity chips on route stops (Mercadona green, Carrefour blue, etc.)

Surface variant `#F2E3D1` (surfaceVariant): input backgrounds in the weight sliders on RouteScreen, OCR item edit fields.

## Screen-by-Screen Interaction Contract

### RouteScreen (`frontend/src/screens/lists/RouteScreen.tsx`)

**Trigger:** User taps "Optimizar ruta" — sends `POST /optimizer/optimize/` with user location, `max_stops`, and weight preferences.

**Loading state:** Replace mock result with `SkeletonBox` rows (3 rows × 56px height, 8px gap) while request is in flight. Skeleton uses `surfaceVariant` as shimmer base.

**Success state:**
- Hero card: `priceLarge` total price (accent color), `textStyles.caption` subtitle "Precio total estimado"
- Savings badge: `success` color, "Ahorrás X €" label, `textStyles.label`
- Stop list: ordered route stops in `listItemHeight: 56` rows with chain color indicator dot (6px diameter, pill radius), store name in `textStyles.body`, distance + time in `textStyles.caption` + `textMuted`
- "Ver en mapa" secondary button below stop list (not primary accent — use outlined/ghost variant)

**Error state (OPTIMIZER_NO_STORES_IN_RADIUS):** Full-width error card, `errorBg` bg, `error` icon, message copy (see Copywriting), "Ampliar radio" button in accent primary style.

**Weight configuration:** "Ajustar preferencias" tappable row above the CTA opens `AppModal` with three sliders (Precio 0–100, Distancia 0–100, Tiempo 0–100). Sliders sum normalizes internally — no UI enforcement needed. Confirm button closes modal and retriggers optimize if values changed.

**max_stops selector:** Segmented control or stepper: 2 / 3 / 4 / 5 stops. Default selection: 3. Use `surface` background, accent for selected segment.

### OCRScreen (`frontend/src/screens/lists/OCRScreen.tsx`)

**Trigger:** "Escanear lista" — opens `expo-image-picker` (camera or gallery). On image selection, sends `POST /ocr/scan/` as `multipart/form-data`.

**Loading state:** Full-screen `SkeletonBox` overlay (or spinner) while OCR processes. Copy: "Procesando imagen..." in `textStyles.body` centered below skeleton.

**Success state:** Scrollable list of recognized items. Each row:
- Left: raw OCR text in `textStyles.bodySmall` + `textMuted`
- Right: matched product name in `textStyles.body` (or "Sin coincidencia" in `textDisabled` if unmatched)
- Confidence badge: `accent` bg (>= 80% match) or `surfaceVariant` bg (< 80%), `textStyles.label`
- Quantity stepper: minus / number / plus inline controls, height 32px
- Unmatched rows: user can type a product name in an inline text input

**Empty state (no items recognized):** See Copywriting contract.

**CTA:** "Añadir a mi lista" — adds all checked items. Unchecked items are excluded. Default: all matched items pre-checked; unmatched items pre-unchecked.

### AssistantScreen (`frontend/src/screens/assistant/AssistantScreen.tsx`)

**Trigger:** User types message in bottom input and taps send icon (Ionicons `send` icon, accent color).

**Loading state:** Typing indicator — three animated dots in a left-aligned assistant bubble (surface bg). Appears immediately on send, replaced by actual response.

**Message display:**
- User messages: right-aligned bubble, `surface` bg, `textStyles.body`, `text` color. Max width: 75% of screen.
- Assistant messages: left-aligned bubble, `surfaceVariant` bg, `textStyles.body`, `text` color. Max width: 85% of screen.
- Bubble padding: `md` (16px) horizontal, `sm` (8px) vertical. Border radius: `borderRadius.lg` (20px) with 4px on the "tail" corner.

**Guardrail rejection message:** Displayed as a regular assistant bubble — does NOT use error color. Copy in Copywriting contract.

**Empty state (no messages):** See Copywriting contract.

**Input bar:** Bottom-anchored, `surface` bg, top border `border` color. TextInput + send button. Min height: 44px. Respects keyboard avoidance.

## Interaction States

Each interactive element must implement all four states. Use the established `colors.*` tokens:

| State | Visual treatment |
|-------|-----------------|
| Default | As specified per element |
| Pressed / active | `primaryDark` (#C03E0E) for accent elements; `surfaceVariant` for surface elements. Opacity 0.85 on non-color elements. |
| Disabled | `textDisabled` (#B0AFA6) text, `surfaceVariant` bg. Not interactive. |
| Loading (in-flight request) | `SkeletonBox` shimmer for content areas. Primary CTA: show activity indicator (white spinner) replacing label text. Button remains disabled during request. |
| Error | `errorBg` card with `error` (#C0392B) icon + message + recovery CTA. |

## Accessibility

- All touch targets: minimum 44×44px (enforced by `sizes.buttonHeight.md: 48` and `sizes.searchBarHeight: 44`)
- Text contrast: all body text (`#1A1A18` on `#FAFAF7`) meets WCAG AA (contrast ratio > 7:1)
- Accent on white (`#E8541A` on `#FAFAF7`): contrast ratio ~3.5:1 — use for large text (18px+) or decorative only; pair with `primaryDark` (#C03E0E) for small interactive labels if needed
- `accessibilityLabel` required on: send icon button (AssistantScreen), OCR scan button, route optimize button, chain color indicator dots (use store name as label), confidence badges
- Chat scroll: use `FlatList` with `inverted` prop for chat message list (AssistantScreen). Keyboard avoidance via `KeyboardAvoidingView`.
- OCR item list: each row must be accessible as a single unit with `accessibilityRole="checkbox"` for the check state
